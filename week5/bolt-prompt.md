# Bolt Prompt for Version 1

Build a responsive CRUD web app called Study Task Board.

The app should help a student manage coursework tasks. Users need to create, read, update, and delete tasks. Each task should have:
- title, required, 3 to 80 characters
- category with options Reading, Coding, Writing, Review
- priority with options Low, Medium, High
- notes, optional, up to 220 characters
- done status, so the user can complete or reopen the task

Use a simple but polished UI with a task form, summary counts for total and open tasks, task cards, edit/delete/complete controls, and filters for all/open/done. Persist tasks locally in the browser using localStorage. Include basic validation and clear error messages. Keep the app easy to run locally with no backend.

Also generate a README with setup, run instructions, persistence notes, known issues, and manual fix notes.
